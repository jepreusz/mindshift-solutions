


Known Issue

BIG-IP virtual addresses send gratuitous Address Resolution Protocol (ARP) messages after a failover event when ARP is disabled.
This issue occurs when all of the following conditions are met:

ARP is disabled for a virtual address.

A failover event to another device in the device group occurs.


Impact

BIG-IP virtual addresses that should not respond to ARP requests start receiving traffic after failover.

Symptoms

As a result of this issue, you may encounter the following symptoms:

BIG-IP iHealth lists Heuristic H435082 on the Diagnostics > Identified > Medium screen.
Virtual addresses with ARP disabled send gratuitous ARP messages on failover.


ResolutionStatus

F5 Product Development has assigned ID 432720 to this issue. F5 has confirmed that this issue exists in the products listed in the Applies To box, located in the upper-right corner of this article. For information about releases or hotfixes that resolve this issue, refer to the following table:

Workaround

To work around this issue, disable the ICMP ECHO option. To disable ICMP ECHO, perform the following procedure:
Impact of workaround: Performing the following procedure should not have a negative impact on your system.

Using the BIG-IP Configuration utility to disable ICMP ECHO


Log in to the BIG-IP Configuration utility as an administrative user.
Navigate to Local Traffic > Virtual Servers > Virtual Address List.

Click the name of the virtual address you want to modify.
Clear the ICMP ECHO check box.
Click Update.


Using the tmsh utility to disable ICMP ECHO


Log in to the Traffic Management Shell (tmsh) by typing the following command:tmsh

To disable ICMP ECHO for a specific virtual address, type the following command, where <virtual-address> is the name of the virtual address that you want to modify:modify /ltm virtual-address <virtual-address> icmp-echo disabled

Save the change by typing the following command:save /sys config
