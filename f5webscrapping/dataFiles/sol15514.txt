


Known IssueThe memory usage on a standby BIG-IP system may continuously increase. This issue occurs when all of the following conditions are met:Your system uses a diameter virtual server with the Connection Mirroring setting enabled.The diameter virtual server is associated with a diameter profile that has the following options set to a value greater than zero (0):	Maximum Retransmit Attempts (default value is 1)Retransmit Timeout (default value is 10)	ImpactOver time, standby unit performance may degrade, and the standby unit may eventually become unresponsive or reboot due to memory exhaustion.SymptomsAs a result of this issue, you may encounter the following symptoms:BIG-IP iHealth lists Heuristic H475795 on the Diagnostics > Identified > High  screen.Memory usage on the standby unit continuously increases.The standby unit stops responding or reboots due to memory exhaustion.ResolutionStatusF5 Product Development has assigned ID 461350 to this issue. F5 has confirmed that this issue exists in the products listed in the Applies To box, located in the upper-right corner of this article. For information about releases or hotfixes that resolve this issue, refer to the following table:Type of FixFixes Introduced InRelated ArticlesRelease11.6.0			11.5.2SOL2200: Most recent versions of F5 softwareHotfix11.5.0 HF4			11.4.1 HF6SOL9502: BIG-IP hotfix matrixWorkaroundTo work around this issue, you can disable connection mirroring on the affected diameter virtual server. Alternatively, you can disable diameter retransmission for the affected diameter virtual server by configuring the Maximum Retransmit Attempts setting and the Retransmit Timeout setting in the associated diameter profile to 0. To do so, perform either of the following procedures:Disabling the Connection Mirroring setting on the diameter virtual serverImpact of workaround: Performing the following procedure should not have a negative impact on your system.Log in to the Configuration utility.Navigate to Local Traffic > Virtual Servers > Virtual Server List.Click the name of the affected diameter virtual server.Clear the Connection Mirroring check box.To save the changes, click Update.Ensure that the configuration is synchronized between the high availability (HA) units by performing the appropriate procedure described in SOL13920: Performing a ConfigSync using the Configuration utility.Disabling diameter retransmission for the diameter virtual serverImpact of workaround: Performing the following procedure should not have a negative impact on your system.Log in to the Configuration utility.Navigate to Local Traffic > Profiles > Services > Diameter.Click the name of the diameter profile associated with the affected diameter virtual server.Select the Maximum Retransmit Attempts check box if it is not already selected, and set the value to zero (0).Select the Retransmit Timeout check box if it is not already selected, and set the value to zero (0).To save the changes, click Update.Ensure that the configuration is synchronized between the HA units by performing the appropriate procedure described in SOL13920: Performing a ConfigSync using the Configuration utility.Supplemental InformationSOL4918: Overview of the F5 critical issue hotfix policySOL167: Downloading software and firmware from F5SOL13123: Managing BIG-IP product hotfixes (11.x - 12.x)
